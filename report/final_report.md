# Финальный отчёт. Прогноз отклика клиента на оформление банковского депозита

**Автор:** Байбакова Екатерина  
**Курс:** ML-base  
**Проект:** прогноз отклика клиента на оформление банковского депозита

## 1. Введение и постановка задачи

В проекте решается задача бинарной классификации в банковском маркетинге.

Нужно предсказать, оформит ли клиент срочный депозит, на основе информации о клиенте, финансовых признаков и истории маркетинговых контактов.

Целевая переменная:

- `yes` — клиент оформил депозит;
- `no` — клиент не оформил депозит.

Практический смысл задачи: банк может заранее выбрать клиентов с большей вероятностью отклика и эффективнее распределить ресурсы колл-центра.

Основная метрика — **PR-AUC**.

PR-AUC выбрана как главная метрика, потому что классы несбалансированы: клиентов с положительным откликом заметно меньше. В такой ситуации accuracy может быть обманчивой.

Дополнительные метрики:

- F1-score;
- precision;
- recall;
- ROC-AUC.

## 2. Поиск и описание данных

Используется датасет Bank Marketing UCI Dataset.

Основной файл:

```text
data/raw/bank-full.csv
```

Размер датасета:

```text
45211 строк, 17 исходных колонок
```

Данные описывают прямые маркетинговые кампании банка. В них есть признаки клиента, финансовые признаки, признаки текущего контакта, история прошлых маркетинговых контактов и целевой признак `y`.

## 3. Обработка и подготовка данных

Были выполнены:

- проверка размера датасета;
- проверка типов данных;

- проверка дублей;

Разбиение данных:

```text
train: 27126 объектов
validation: 9042 объекта
test: 9043 объекта
```

## 4. Baseline-модель

В качестве baseline использовался `DummyClassifier`, который предсказывает самый частый класс.

Результат baseline:

```text
PR-AUC = 0.117
F1 = 0.000
ROC-AUC = 0.500
```

Baseline нужен как нижняя граница качества.

## 5. Эксперименты

Были обучены и сравнены:

- LogisticRegression;
- RandomForest;
- ExtraTrees;
- GradientBoosting;
- HistGradientBoosting;
- VotingEnsemble.

Для моделей использовался GridSearchCV с основной метрикой PR-AUC.

Краткие результаты на validation split:

| Модель | PR-AUC | Комментарий |
|---|---:|---|
| HistGradientBoosting | 0.451 | лучшая модель по PR-AUC |
| GradientBoosting | 0.444 | близкий результат |
| RandomForest | 0.443 | хороший F1, но ниже PR-AUC |
| VotingEnsemble | 0.441 | ансамбль не превзошёл лучший бустинг |
| ExtraTrees | 0.428 | хуже бустингов |
| LogisticRegression | 0.408 | базовая линейная модель |
| DummyClassifier | 0.117 | baseline |

## 6. Финальная модель и интерпретируемость результатов

Финальная модель:

```text
HistGradientBoosting
```

Лучшие параметры:

```text
learning_rate = 0.03
max_iter = 220
max_leaf_nodes = 15
```

Финальный threshold:

```text
0.21
```

Результаты на test split:

| Модель | Threshold | F1 | Precision | Recall | ROC-AUC | PR-AUC |
|---|---:|---:|---:|---:|---:|---:|
| HistGradientBoosting | 0.21 | 0.504 | 0.478 | 0.532 | 0.806 | 0.481 |

Интерпретация:

- ROC-AUC около 0.806 показывает хорошее разделение классов.
- PR-AUC около 0.481 существенно выше baseline около 0.117.
- Recall около 0.532 означает, что модель находит примерно половину клиентов с положительным откликом.
- Precision около 0.478 означает, что почти половина клиентов, отмеченных моделью как перспективные, действительно относится к положительному классу.
